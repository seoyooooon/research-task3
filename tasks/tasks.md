# 1, 2차 과제들 기존에 걸렸던 시간 및 완성도 부분에서 개선되는지 확인

## [1차 과제] 카메라를 이용한 제스처 인식 기술 제안

### (1) 시간 측면
| 구분 | 총 개발 소요 시간 |
| -- | -- | 
| ChatGPT 사용 | 약 6시간 |
| Antigravity(AI 에이전트) 사용 | 약 5분 |

- 현재 Antigravity한테 준 가이드가 1차 과제 당시 ChatGPT한테 준 가이드보다 훨씬 정돈되어있기는 하지만, 그 차이를 감안하더라도 Antigravity를 사용한 개발은 코드를 바로 실행 가능하도록 만들어주기 때문에 개발 소요 시간이 확연히 줄어들었다.  
- 결과물을 확인하고 수정하고 싶은 사항이 있더라도 원하는 내용을 입력하면 파일에서 직접 바꿔주기 때문에 기존의 ChatGPT를 사용했을 때의 코드 생성 -> 복붙 -> 에러 -> 수정 -> 실행 과정이 생략되어 편리했다. 

### (2) 완성도 측면
1. **제스처 간 명확한 분리**
   - 1차 과제 당시 스냅과 흔들기 제스처 간 오인식이 너무 심해 흔들기 제스처 구현을 결국 포기했었다. 하지만 에이전트는 방향 전환 횟수 계산 및 직선성 비율 필터 등을 통해 두 제스처를 명확히 분리했다. 
   - 지난번에는 구현하지 못했던 가상 TV 채널 화면도 완성도 높은 BGR 애니메이션으로 깔끔하게 구현하여 제스처 조작에 따른 피드백을 시각적으로 편리하게 확인할 수 있게 되었다.

2. **기술 제안서와 실행 메뉴얼 자동 제공**
   - 핵심 실행 코드뿐만 아니라 기술 제안서(`proposal.md`)와 사용 가이드(`README.md`)를 알아서 꼼꼼하게 만들어 주어 최종 결과물의 퀄리티가 크게 향상되었다.


## [2차 과제] 얼굴 가상 시착을 활용한 TV 기능 확장

### (1) 시간 측면
| 구분 | 총 개발 소요 시간 |
| -- | -- | 
| ChatGPT 사용 | 약 8시간 |
| Antigravity(AI 에이전트) 사용 | 약 20분 |

- 기존 ChatGPT를 사용해 2차 과제를 개발할 때는 2D 이미지의 affine 회전 연산 매핑, 웹캠 좌표 변환, 프레임 떨림 제어 필터를 직접 코딩하고 최적값을 찾는 과정에서 6시간 이상의 오랜 시행착오를 겪었다.
- 에이전트를 도입했을 때는 최신 MediaPipe Tasks API 버전 간 임포트 이슈 등의 돌발적인 모듈 에러를 스스로 디버깅하여 해결했을 뿐만 아니라, 누끼 잔상을 없애기 위한 경계면 침식(Erosion) 연산 및 실시간 미세조정 트랙바 튜닝까지 약 30분 만에 원스톱으로 빌드 및 실행을 완료하여 개발 효율이 비약적으로 증가했다.

### (2) 완성도 측면
평가 지표
| 항목 | 점수(10점 만점) |
| -- | -- |
| 얼굴 인식 정확도 | 9점 |
| 시착 자연스러움 정도 | 8점 |
| 조명/환경 영향 대응력 | 8점 |
| 재사용 의향 | 10점 |

1. **AI 에셋 생성 및 정밀 누끼 전처리 자동화**
   - 과거 2차 과제 수행 시에는 구글에서 PNG 안경 이미지를 직접 구해와 사용했다. 반면 에이전트는 '다리가 없는 정면 프레임' 전용 에셋을 즉석 제작하고, 침식 및 여백 자동 크롭 필터를 설계해 외곽의 흰색 테두리 흔적을 제거한 투명 누끼 PNG를 만들어 사용했다. 생성된 사진을 사용했기 때문에 투명 누끼 PNG가 자연스럽지는 않다. 

2. **EMA 필터 및 실시간 캘리브레이션 트랙바 탑재**
   - 랜드마크의 파르르 떨리는 지터링 현상을 제어하기 위해 지수 이동 평균(EMA) 필터를 설계하여 얼굴을 부드럽게 추종하도록 구현했다.
   - 또한 사용자가 실시간으로 화면을 보며 피팅 크기 및 수직 높낮이를 본인 얼굴에 맞게 수정할 수 있는 트랙바 조절 시스템을 탑재해 완성도를 높였다.

3. **한계점 및 향후 보완 과제**
   - 지난 과제의 결과물과 같이 고개가 일정 각도 이상 회전할 경우 안면 랜드마크가 카메라 앵글에서 가려져 안경 매핑의 자연스러움이 감소하는 물리적 한계가 있다. 
   - 또한, 2D 평면 이미지의 특성상 정면 안경 다리가 표현되지 못하는 아쉬움이 있으며, 향후 3D 원근 투영 변환 또는 3D CAD 모델 렌더링을 융합하는 구조적 고도화가 필요하다.

# AI 에이전트 활용 총평
1. 시간 측면에서 압도적인 개발 효율화가 이루어졌다. 코딩 경험이 부족해도 아이디어를 빠르고 정확하게 구현할 수 있다는 점이 가장 큰 장점인 것 같다. 
2. 코딩을 할 때 디버깅하는 데에 시간이 제일 오래 걸리는데 이 과정을 에이전트가 빠르게 처리해주는 것이 엄청 인상깊었다. 
3. 아이디어를 입력하면 Implementation Plan을 만들어서 보여주는데, 구체적으로 코드를 어떻게 만들 예정인지 흐름을 보여주고 허락을 받는 것이 신뢰가 간다. 내가 제안한 내용보다 더 좋은 생각이 있으면 알려주기도 하는 것이 마음에 든다.
